How it works

1

Messages arrive

Every message is stored verbatim in a local SQLite database. Nothing is discarded.

2

Compaction

Older messages are grouped into leaf summaries by a background daemon — your session is never blocked.

3

DAG nodes

Leaf summaries condense further into higher-level nodes, forming a directed acyclic graph of your project history.

4

Cipher promotion

High-signal nodes — architectural decisions, bug fixes, design rationale — are promoted to Cipher, the long-term semantic memory store.

5

Context assembly

Each new session is assembled from recent raw messages + DAG summaries + Cipher recall. Full fidelity, minimal tokens.

Context is never truncated. Each turn, it's assembled fresh — recent raw messages, DAG summaries, and Cipher recall — always within the model's window. Compaction runs automatically in the background after each turn, triggered once enough messages accumulate.

Built on lossless-claw

What we inherited

The DAG-based compaction architecture, lossless message retention, and the LCM memory model — all from lossless-claw by Martian Engineering. We're grateful for their foundational work.

What we added

  • Long-term memory via Cipher — high-signal summaries promoted to persistent semantic storage, recalled across sessions
  • Smart promotion engine — automatically detects architectural decisions, bug fixes, and design rationale worth keeping forever
  • Background daemon — compaction never blocks your session
  • Anthropic-native + OpenAI-compatible — use any summarization model
  • Claude Code native — plugin manifest, lifecycle hooks, MCP tools
